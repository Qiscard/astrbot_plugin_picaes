# 图片加解密插件

一个基于 API 的图片马赛克加密/解密插件，通过像素块打乱实现图片加密，支持自定义加密等级和密钥。

## 功能特性

- ✅ **图片加密**：将图片像素块打乱，生成马赛克密图
- ✅ **图片解密**：使用相同参数将密图还原为原图
- ✅ **10 级加密强度**：从 10×10 到 100×100 像素块，等级越高加密越强
- ✅ **自定义密钥**：支持自定义密钥，默认密钥为 `tool.hadsky.com`
- ✅ **格式自动检测**：自动识别 PNG/JPG/GIF/WebP，保留透明通道
- ✅ **原始字节传输**：绕过平台图片压缩，确保加解密数据一致

## 安装方法

1. 将 `astrbot_plugin_picaes` 文件夹放入 AstrBot 的 `data/plugins` 目录
2. 在 AstrBot 后台插件管理页面重载插件
3. 插件依赖 `aiohttp`，首次加载会自动安装（服务器需能访问外网）

## 配置说明

在 AstrBot 后台 → 插件管理 → astrbot_plugin_picaes → 插件配置 中修改：

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `api_url` | string | `https://picace.995456.xyz/api/proxy` | 加解密 API 接口地址 |
| `default_key` | string | `tool.hadsky.com` | 未指定密钥时的默认密钥 |
| `default_level` | int | `4` | 未指定等级时的默认加密等级（1-10） |
| `timeout` | int | `120` | API 请求超时时间（秒） |

### 加密等级对照

| 等级 | 像素块大小 | 说明 |
|-----|-----------|------|
| 1 | 10×10 | 最低加密，处理最快 |
| 4 | 40×40 | 默认等级，平衡安全与速度 |
| 7 | 70×70 | 较高加密 |
| 10 | 100×100 | 最高加密，处理较慢 |

## 使用命令

### 加密图片

| 命令 | 说明 |
|------|------|
| `加密` + 图片 | 使用默认等级和密钥加密 |
| `加密 6` + 图片 | 指定等级 6 加密 |
| `加密 6 mykey` + 图片 | 指定等级 6，密钥 `mykey` |

### 解密图片

| 命令 | 说明 |
|------|------|
| `解密` + 图片 | 使用默认等级和密钥解密 |
| `解密 6` + 图片 | 指定等级 6 解密 |
| `解密 6 mykey` + 图片 | 指定等级 6，密钥 `mykey` |

### 使用方式

**方式一：** 直接发送图片，然后发送命令

```
[发送/回复一张图片]
加密 7 mysecretkey
```

**方式二：** 回复一张图片并发送命令

```
[回复一张图片]
解密 7 mysecretkey
```

> ⚠️ **重要：** 发送图片时请选择「**文件**」形式发送（不要选相册/预览），否则聊天平台可能会压缩图片导致解密失败。

## 加解密原理

- **加密**：将图片划分为 N×N 的像素块网格，通过密钥生成的随机映射打乱各块位置
- **解密**：使用相同的等级和密钥，执行逆映射将像素块还原到原始位置
- **密钥一致性**：加密和解密必须使用**完全相同的等级和密钥**，否则无法正确还原

## 错误说明

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `请发送图片！` | 消息中未附带图片 | 发送或回复一张图片 |
| `下载图片失败` | 图片 URL 无法访问 | 重新发送图片 |
| `图片解密失败：<API 错误信息>` | API 返回错误 | 检查等级和密钥是否与加密时一致 |
